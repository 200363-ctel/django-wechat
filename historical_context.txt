通常与人的不安全行为、物的不安全状态、环境因素以及管理缺陷有关，而安全检查工作正是识别这些问题并采取预防性措施的重要手段。基于这一理论，本课题将安全检查系统定义为集“检查、识别、记录、整改、反馈、统计”于一体的全过程管理工具。\n其次，在需求提取阶段，对水利水电施工现场常见作业活动及其安全检查要点进行归纳。水利水电施工通常包括基坑开挖、边坡支护、混凝土浇筑、起重吊装、模板支架、高空作业、临时用电及施工机械运行等多个高风险环节。不同环节对应不同的检查标准和风险控制要求，因此系统设计必须体现检查对象的分类管理和检查项目的针对性设置。通过对现场管理流程的分析，可将系统使用者划分为管理员、安全员、检查人员、项目负责人等角色，并根据不同角色赋予差异化操作权限，以保证系统运行的规范性和可控性。\n在系统设计阶段，课题将从系统架构设计、功能模块设计、数据库设计和业务流程设计四个方面展开。系统架构上，可采用前后端分离或传统分层架构思想，划分为表示层、业务逻辑层和数据层。表示层主要负责用户界面展示和操作交互；业务逻辑层负责安全检查业务规则处理，如隐患等级判定、整改期限控制、复查状态更新等；数据层负责各类信息的存储、查询与维护。功能模块方面，系统初步设计为五大模块：一是用户与权限管理模块，用于管理不同用户身份及操作范围；二是安全检查任务管理模块，用于制定检查计划、发布检查任务并记录检查结果；三是隐患整改管理模块，用于对发现的问题进行分类、定级、下发整改要求并追踪整改进度；四是统计分析模块，用于生成隐患数量、隐患类型分布、整改率、超期率等统计结果；五是系统维护模块，用于标准库维护、数据备份和参数设置等。数据库设计方面，围绕“用户表、检查表、隐患表、整改表、复查表、项目表”等核心数据实体建立逻辑关系，保证数据一致性与完整性。\n在理论分析及计算部分，本课题虽然以系统设计为主，但仍可引入一定的数据统计与指标分析方法，用于支持系统功能合理性。例如，可设置隐患整改率=已完成整改隐患数/隐患总数×100%，隐患复查通过率=复查合格数/复查总数×100%，逾期整改率=逾期未整改隐患数/隐患总数×100%。通过这些指标计算，可以反映施工现场安全管理执行效果，也为后续统计分析模块的设计提供依据。若条件允许，还可对不同施工阶段的隐患分布进行对比分析，以判断系统在风险预警和管理改进方面的辅助价值。\n在实验方法和步骤方面，本课题主要采用模拟业务场景验证的方式开展。第一步，整理水利水电施工安全检查业务流程和典型检查表内容，形成系统需求清单。第二步，根据需求清单绘制系统功能结构图、业务流程图和数据库E-R图，完成系统总体设计。第三步，选取典型场景，如日常巡检、专项检查、隐患整改通知、复查闭环等，对系统各模块进行流程模拟，验证功能衔接是否合理。第四步，通过构造测试数据，检验系统对检查记录录入、隐患信息分类、整改状态更新和统计报表生成等功能的支持情况。第五步，对系统设计中存在的问题进行修改优化，如功能重复、流程不清晰、字段设置不完整等。第六步，总结系统设计成果，并从实用性、规范性、可扩展性和安全管理效能提升等角度进行综合评价。\n总体来看，本课题方案兼顾了安全工程专业背景下的风险管理需求和计算机软件类系统设计要求，能够较好体现理论联系实际的研究特点。通过该总体方案的实施，预期形成一套逻辑清晰、结构完整、适应水利水电施工现场安全管理需要的安全检查系统设计成果。\n\n三、项目进展、存在问题及解决办法。\n1.毕设工作情况\n本课题目前未更改题目，仍按照“水利水电施工建设中安全检查系统的设计”开展研究。开题以来，已按学校和学院要求推进毕业设计相关工作，围绕课题方向查阅了相关文献资料，并根据毕业论文写作规范逐步完善研究内容。周进展记录已按照要求进行整理，后续将继续及时补充并上传相关材料，确保过程管理资料完整规范。\n2.课题研究进展\n目前已基本完成课题前期调研和文献收集工作，对水利水电施工建设过程中的主要安全风险、安全检查流程及安全信息化管理需求有了较为系统的认识。通过查阅施工安全管理、安全检查系统设计、隐患排查治理机制等相关资料，已明确系统研究的基本思路和总体方向。在研究方法方面，已初步掌握文献分析、功能需求分析、系统模块划分及数据库逻辑设计等方法，并完成了系统功能框架的初步构思。现阶段已对系统涉及的主要业务流程进行了梳理，包括检查任务下达、现场检查记录、隐患上报、整改落实、复查销项和统计分析等内容。论文写作方面，已完成绪论部分初稿，并对第二章相关理论基础和研究方法部分进行了资料整理。\n3. Laboratory of Hydroscience and Engineering, Tsinghua University, Haidian, Beijing 100084, China\n2    China Yangtze Three Gorges Group Corporation, Beijing 100038, China\nCorrespondence should be addressed to Peng Lin; celinpe@tsinghua.edu.cn\nReceived 13 April 2014; Revised 18 June 2014; Accepted 24 June 2014; Published 9 July 2014\nAcademic Editor: Xiao-Wei Ye\n\nAbstract\nThe concern for workers’ safety in construction industry is reflected in many studies focusing on static safety risk identification and assessment. However, studies on real-time safety risk assessment aimed at reducing uncertainty and supporting quick response are rare. A method for real-time safety risk assessment (RTSRA) to implement a dynamic evaluation of worker safety states on construction site has been proposed in this paper. The method provides construction managers who are in charge of safety with more abundant information to reduce the uncertainty of the site. A quantitative calculation formula, integrating the influence of static and dynamic hazards and that of safety supervisors, is established to link the safety risk of workers with the locations of on-site assets. By employing the hidden Markov model (HMM), the RTSRA provides a mechanism for processing location data provided by the real-time location system (RTLS) and analyzing the probability distributions of different states in terms of false positives and negatives. Simulation analysis demonstrated the logic of the proposed method and how it works. Application case shows that the proposed RTSRA is both feasible and effective in managing construction project safety concerns.\n1. Introduction\nThe construction industry is one of the most dangerous industrial sectors worldwide [1, 2]. The working environment and the work tasks themselves are complex. The number of workers on site generally is large. Heavy machinery and a multitude of pipes, materials, and cables are always in evidence. In addition, sites are frequently not “tidy”; it is not surprising that both the fatality and badly injured accident rates on construction site are higher than those in other areas of work [3, 4]. According to the Bureau of Labor statistics in the United States [5], 4253 construction workers died on sites between 2008 and 2012. In China, the official statistics show that, in the one year, 2007, construction industrial accidents caused the deaths of 2722 workers [6]. Hydropower projects, in particular, have been found to have tougher working con- ditions than other construction sites and safety management of such construction projects is more difficult than that in any other segment of the industry [7]. Currently, China has been continuously developing large hydropower projects on main rivers, such as the Jinsha, Nujiang, and Yarlung\nTsangpo River, investing large financial sums and manpower resources [8, 9]. Hence, without the improvement of the safety management, culture of the construction industry and the implementation of the national policy, which is “safety first and prevention oriented” [10] and which is also a vital management goal of hydropower enterprises in China, will not be achieved and it is unlikely that the accident rate on construction sites will decrease.\nRecent studies related to construction safety manage- ment have asserted that most accidents on sites could have been reduced and some even eliminated, if there existed an effective and consistent safety management process of identification, planning, education/training, and inspection [11]. Safety risk assessment is very important for developing a safety management system [12–14]. Numerous risk assess- ment methods, such as safety checklists, fault tree analysis (FTA), and likelihood exposure consequences (LEC), have been developed to assist construction engineers and project managers in safety risk management [15]. The LEC method is widely used in casualty risk assessment in industrial construction [16]. Risk is considered to be the product ofthree factors: risk likelihood/probability (L), risk exposure frequency (E), and consequences (c). Casanovas et al. [17] presented a risk index which included 19 construction work activities and used the product of consequence and proba- bility to obtain the risk value of each activity. In addition, accident statistics, event tree analysis, and failure mode and effects analysis are also applied in safety management in the construction industry [18]. All the above methods, however, produce qualitative or semiqualitative analysis results. Such results cannot be considered reliable. For example, all of the three factors of LEC method are based on the subjec- tive evaluation of experts. Differences in the experience of different experts may lead to different evaluation results of the same risk [16]. In addition, most of these methods are passive approaches, heavily dependent on historical data and lacking timeliness [17, 19]. As a consequence, these methods, although producing results of interest and for guidance, cannot be used to quantitatively assess the current status of safety performance and the risk value.\nIn order to operate a safety risk assessment procedure accurately, some quantitative methods based on probability theory and fuzzy sets theory are proposed. Using machine learning algorithms, Matı,as et al. [20] performed the identi- fication of hazardous sources on construction sites and the subsequent determination of the relationship between the accident and the cause. Matı,as et al. [21] designed a Bayesian network (BN) based method designed to analyze workplace accidents caused by falls from heights. Leu and Chang [22] successfully combined probabilistic methods with traditional methods (event tree analysis), enabling the setting up of a complete logic model to build a quantitative risk assessment method for construction projects. Regarding fuzzy set theory, Lee and Halpin [23] proposed a quantitative approach based on fuzzy logic for measuring the effects of accidents. Nunes [24] developed an expert system aimed at supporting risk analysis based on fuzzy sets. Azadeh et al. [25] developed a fuzzy expert system for performance assessment of health, safety, environment, and ergonomic system factors in a gas refinery with the objectives ofthe reduction of human error, creation of expert knowledge, and interpretation of large amounts of vague data. These quantitative methods based on probability and fuzzy sets theory offer relatively accu- rate evaluation results compared to those of the traditional methods and have been implemented in practical projects. Nevertheless, none of the above methods have been able to adapt to the dynamic context of construction projects, particularly those complex activities which contribute greatly to the safety risk of on-site workers. In addition, such methods cannot provide real-time information about safety states, which is a necessity to enable timely response and rescue.\nRecently, extensive studies of real-time risk assessment have been carried out. An example is provided by the hidden Markov model (HMM) which is effective in many temporal and real-time pattern recognition applications, such as human action recognition [26, 27], speech recognition [28], and prediction of unknown events [29, 30]. Real-time risk assessment methods based on the HMM focus on the field of network security. These methods use the HMM toprocess the observation symbol sequences of invasions and attacks to determine the real-time safety risk of a network [31– 33]. In principle, the HMM based risk assessment method is accessible, but real-time safety risk assessment studies in construction management have not been found in the literature. The main reason for this could be the lack of availability of on-site asset real-time location information.As a consequence, some studies have focused on the development of intelligent control systems [34–37], adopting advanced communication technologies [38, 39], real-time location [40, 41], and intelligent algorithms as the means of developing new automated management and monitoring systems  [42–44]. However, these particular studies con- centrate more strongly on technical performances, such as accuracy of positioning and optimization of communi- cation frequency, while significant features such as safety management functions are not directly targeted. Geofencing which deals with the automatic detection of the entrance of individuals into dangerous preset forbidden zones is the basic safety management model revealed in the literature [39, 40]. But the simple trigger logic of the geofencing can only offer managers poor binary information concerning on- site workers’ behaviour (either within the forbidden zones or not in the forbidden zones) and may lead to frequent false positives. As a consequence, the development of an integrated method to perform real-time safety risk assessment of on- site workers based on probability concepts is necessary. The workers’ real-time location data should be provided by real-time location technologies. Overall, the development of automatic monitoring technology, especially the real-time location technology for individuals, provides great feasibility for implementation of real-time safety risk assessment. And the real-time location system for personnel and the moni- toring system for structure constitute a complete solution for health and safety management in industrial construction.\nIn this study, GPS and Wi-Fi locating technology were chosen as the fundamental tracking tools. HMM was used as the risk assessment algorithm in constructing a RTSRA method for the safety management of large hydropower construction projects. The rest of this paper is organized as follows. Section 2 describes the construction of an assessment framework, including that of the real-time tracking system and the HMM modelling process. In Section 3, a simulation analysis is carried out on safety risk assessment process. Section 4 gives a preliminary application case of a dam construction site.\n2. Unsafe worker activities are direct causes of accidents. Different unsafe activities are usually associated with specific hazards. It has been concluded by Casanovas et al.  [17] that there are 19 health and safety risks found in construction work associated with different activities. According to hazard kinetic characteristics, these risks can be divided into dynamic and static groups (Table 1).\nMost dynamic hazards on the construction site are caused by vehicles, machinery, and risky tools. A static hazard andts affected area remain unchanged for a relatively long period. These latter hazards consist of elevation changes, hazardous installations, and other dangerous zones. For both dynamic and static hazards, the critical measurable factor is the distance between the workers and hazards [17]. Accidents are more likely to happen, if workers are close to hazards. A dissimilarity is that the position of a static hazard is fixed while a dynamic hazard is always moving. In consequence, in addition to the tracking of workers, dynamic hazards must also be monitored by the RTLS in order to make valid safety risk assessments.\nAnother critical factor is the presence of safety supervi- sors. They are responsible for safety management and possess specialized safety knowledge [47]. Effective safety supervi- sion improves the safety performance of workers. The safety supervisor must be in such a position that he can have a posi- tive impact on a worker behaviour [48]. The distance between worker and safety supervisor can be obtained from the RTLS and hence is another measurable factor. 展示所有检查任务，包括任务名称、检查类型、计划时间、执行状态\\n- 支持按状态筛选：待执行、进行中、已完成\\n- 支持按检查类型筛选：日常巡检、专项检查\\n\\n**创建检查任务**\\n- 填写任务名称、检查类型、检查区域、计划时间\\n- 选择检查人员\\n- 选择检查项目（从检查标准库中选择）\\n- 提交后自动通知检查人员\\n\\n**检查记录详情**\\n- 展示检查任务基本信息\\n- 展示检查项目及检查结果\\n- 展示发现的隐患列表\\n- 支持导出检查报告\\n\\n**现场检查录入**\\n- 按检查项目逐项记录检查结果：合格、不合格\\n- 对不合格项填写问题描述\\n- 拍照上传现场照片\\n- 判定隐患等级：一般隐患、较大隐患、重大隐患\\n- 提交检查记录\\n\\n#### 3.2.4 隐患管理\\n\\n**隐患列表**\\n- 展示所有隐患，包括隐患编号、隐患描述、隐患等级、整改状态、责任人\\n- 支持按状态筛选：待整改、整改中、待复查、已闭环\\n- 支持按隐患等级筛选\\n- 支持按时间范围筛选\\n\\n**隐患详情**\\n- 展示隐患基本信息：编号、描述、等级、发现时间、发现人\\n- 展示现场照片\\n- 展示整改要求、整改期限、责任人\\n- 展示整改记录：整改措施、整改照片、完成时间\\n- 展示复查记录：复查结果、复查意见、复查时间\\n\\n**隐患上报**\\n- 填写隐患描述\\n- 选择隐患类型：高处坠落、物体打击、机械伤害、触电、坍塌、其他\\n- 判定隐患等级\\n- 拍照上传现场照片\\n- 选择隐患位置（支持地图定位）\\n- 提交后自动生成隐患编号\\n\\n**整改通知**\\n- 填写整改要求\\n- 设置整改期限\\n- 指定责任人\\n- 发送通知后责任人收到消息提醒\\n\\n**复查记录**\\n- 查看整改前后对比照片\\n- 填写复查意见：合格、不合格\\n- 对不合格项说明原因并要求继续整改\\n- 对合格项确认闭环\\n\\n#### 3.2.5 统计分析\\n\\n**数据概览**\\n- 展示本月检查次数、发现隐患数、整改完成数、逾期未整改数\\n- 展示隐患整改率、复查通过率、逾期整改率\\n\\n**隐患分布统计**\\n- 按隐患类型统计数量，以饼图展示\\n- 按隐患等级统计数量，以柱状图展示\\n- 按施工区域统计数量，以列表展示\\n\\n**整改进度统计**\\n- 展示各责任人的隐患整改情况\\n- 展示逾期未整改隐患列表\\n- 支持按时间范围查询\\n\\n#### 3.2.6 系统管理\\n\\n**用户管理**\\n- 展示所有用户列表\\n- 支持添加、编辑、删除用户\\n- 用户信息包括：姓名、工号、手机号、所属单位、角色\\n\\n**权限设置**\\n- 为不同角色分配操作权限\\n- 管理员：全部权限\\n- 安全员：检查任务管理、隐患审核、统计查询\\n- 检查人员：检查记录录入、隐患上报\\n- 项目负责人：统计查询、整改监督\\n\\n**检查标准库**\\n- 维护检查项目分类：基坑开挖、边坡支护、混凝土浇筑、起重吊装、模板支架、高空作业、临时用电、施工机械\\n- 为每个分类配置具体检查项目\\n- 支持添加、编辑、删除检查项目\\n\\n#### 3.2.7 个人中心\\n\\n**个人信息**\\n- 展示用户姓名、工号、手机号、所属单位、角色\\n- 支持修改手机号\\n\\n**消息通知**\\n- 展示系统消息：检查任务通知、整改通知、复查通知\\n- 支持标记已读\\n\\n**设置**\\n- 修改密码\\n- 退出登录\\n\\n## 4. 业务规则与逻辑\\n\\n### 4.1 检查任务流程\\n1. 安全员创建检查任务并指定检查人员\\n2. 检查人员收到任务通知\\n3. 检查人员到现场执行检查，逐项记录检查结果\\n4. 发现不合格项时判定为隐患并上报\\n5. 检查完成后提交检查记录\\n6. 安全员查看检查记录并审核\\n\\n### 4.2 隐患整改流程\\n1. 检查人员或安全员上报隐患\\n2. 安全员审核隐患信息并下发整改通知\\n3. 责任人收到整改通知\\n4. 责任人完成整改并上传整改照片\\n5. 安全员进行现场复查\\n6. 复查合格则隐患闭环，不合格则继续整改\\n\\n### 4.3 隐患等级判定规则\\n- 一般隐患：可能导致轻伤或轻微财产损失\\n- 较大隐患：可能导致重伤或较大财产损失\\n- 重大隐患：可能导致死亡或重大财产损失\\n\\n### 4.4 整改期限规则\\n- 一般隐患：整改期限不超过7天\\n- 较大隐患：整改期限不超过3天\\n- 重大隐患：立即整改，整改期限不超过24小时\\n\\n### 4.5 逾期提醒规则\\n- 整改期限到期前1天，系统自动向责任人发送提醒消息\\n- 整改期限到期后，系统自动标记为逾期并向安全员和项目负责人发送通知\\n\\n### 4.6 数据统计规则\\n- 隐患整改率 = 已完成整改隐患数 / 隐患总数 × 100%\\n- 复查通过率 = 复查合格数 / 复查总数 × 100%\\n- 逾期整改率 = 逾期未整改隐患数 / 隐患总数 × 100%\\n\\n## 5. 课题主要研究内容\\n本课题围绕水利水电施工建设过程中的安全管理需求，设计一套适用于施工现场的安全检查系统。水利水电工程通常具有施工周期长、作业环境复杂、交叉作业频繁、高空作业与临时用电风险突出等特点，传统依赖人工巡查和纸质记录的安全检查方式存在信息滞后、隐患追踪困难、整改闭环不完善等问题。因此，本课题拟结合安全工程理论、施工现场风险管理需求以及信息化系统设计方法，对水利水电施工建设中的安全检查流程进行系统研究与优化。\\n首先，将分析水利水电施工现场常见的安全风险类型，包括高处坠落、物体打击、机械伤害、触电、坍塌、有限空间作业风险等，并结合相关安全生产法律法规、行业标准和施工管理制度，明确安全检查系统应具备的基本功能和应用目标。其次，围绕施工单位、管理人员和检查人员的实际使用需求，对系统的功能模块进行划分，重点设计用户管理、隐患排查、检查任务发布、隐患整改跟踪、统计分析、信息查询和预警提示等功能。再次，在系统设计层面，课题将从总体架构、数据库结构、功能流程、界面逻辑等方面展开分析，形成较为完整的系统方案。\\n最后，本课题还将通过业务流程梳理和模块设计，对系统运行过程中的数据流转关系进行分析，建立从“检查—记录—整改—复查—归档”的安全管理闭环机制。最后，结合水利水电施工场景，对所设计系统的适用性、可行性和预期应用效果进行评价，论证该系统在提升施工安全管理效率、规范隐患治理流程、降低事故风险方面的实际意义，为水利水电工程施工安全信息化建设提供一定参考。\\n2、 课题与安全工程专业契合度描述\\n本课题与安全工程专业具有较高的契合度，体现在研究对象、研究目的、研究方法以及成果应用等多个方面。安全工程专业的核心任务是识别危险源、分析事故致因、控制安全风险、预防生产事故，并通过工程技术和管理手段提升系统本质安全水平。本课题以水利水电施工建设中的安全检查系统设计为研究对象，本质上就是围绕施工安全风险防控和隐患治理机制优化展开研究，符合安全工程专业的培养方向和研究范畴。\\n首先，从研究目标来看，本课题旨在解决施工现场安全检查信息化水平不足、隐患整改跟踪不及时、检查记录管理不规范等现实问题，这些问题直接关系到施工安全管理质量和事故预防效果。通过设计安全检查系统，可以强化隐患排查治理的闭环管理，提高风险识别与动态控制能力，这与安全工程专业强调的“预防为主、风险管控、持续改进”的理念高度一致。其次，从专业知识运用来看，课题需要综合运用安全系统工程、安全管理学、事故致因理论、危险源辨识、风险评价、职业健康安全管理等专业知识，对施工现场的风险特点和安全管理流程进行分析，并将这些理论要求转化为系统功能设计依据。\\n再次，从工程应用角度看，水利水电工程属于高风险建设领域，安全管理要求高、涉及面广，是安全工程专业典型的应用场景之一。课题研究结果不仅能够服务于具体施工项目的安全检查管理，还能够为类似大型土木工程、基础设施建设项目的安全信息化管理提供借鉴。最后，从毕业论文培养目标来看，本课题兼具理论分析与工程实践特征，能够有效锻炼问题分析、方案设计、系统构建和专业表达等方面的综合能力，体现了安全工程专业“工程背景突出、管理技术结合、服务安全生产实际”的人才培养要求。因此，本课题在选题意义、研究内容、专业方法和应用价值上均与安全工程专业紧密相关。\\n\\n\\t二、研究方案设计\\n1、研究方法\\n本课题采用文献研究法、系统分析法、需求分析法、功能模块设计法以及数据库设计方法相结合的研究思路，对水利水电施工建设中安全检查系统进行设计。首先，在课题研究初期，采用文献研究法查阅国内外关于施工安全管理、安全检查信息化系统、隐患排查治理机制以及水利水电工程施工安全控制等方面的研究成果，系统了解当前研究现状、常用技术路线及实际应用模式。通过对相关法律法规、国家标准、行业规范和已有研究文献的整理分析，明确系统设计必须满足的规范要求和业务逻辑基础。\\n其次，采用系统分析法对水利水电施工现场安全检查业务进行分解，从管理对象、管理流程、风险要素、参与角色和信息传递路径等多个维度梳理系统运行机制。该方法有助于从整体角度把握安全检查工作的结构组成，避免功能设计碎片化，使系统能够覆盖检查任务制定、现场检查实施、隐患记录、整改通知、复查验收、统计汇总等完整流程。由于水利水电施工现场涉及建设单位、监理单位、施工单位及班组等多主体协同，系统分析法有利于明确各方权限边界与信息交互关系。\\n再次，采用需求分析法确定系统功能。需求分析主要从实际应用出发，围绕安全管理人员、检查人员和项目负责人等不同用户群体的使用需求，分析他们在日常管理中面临的问题，如记录不统一、隐患难追踪、信息统计困难、整改时效性差等，并据此提炼出系统应具备的核心需求，包括账户权限管理、检查项目配置、隐患信息录入、整改通知下发、复查结果记录、报表统计分析和历史数据查询等功能。\\n在系统实现思路方面，课题将采用功能模块设计法，将复杂系统划分为相对独立又相互关联的模块，包括基础信息管理模块、安全检查模块、隐患整改模块、统计分析模块和系统维护模块。通过模块化设计，不仅可以使系统结构更加清晰，便于后续功能扩展，也符合软件系统设计的基本原则。同时，在数据层面采用数据库设计方法，对用户信息、检查记录、隐患信息、整改记录、复查结果等关键数据进行逻辑建模，确保数据存储规范、查询高效、关联清晰。\\n综合来看，这些方法的选择依据在于：一方面，本课题不仅需要符合安全工程专业对风险管理和事故预防的理论要求，也需要满足计算机软件类课题对系统逻辑性和可实现性的要求；另一方面，所选方法能够从理论支撑、业务分析、功能构建和数据组织四个层面共同服务于系统设计目标，保证研究结果既有专业性，也具有较强的工程应用价值。\\n\\n2、总体方案（包括理论分析及计算、实验方法和步骤）\\n本课题总体方案按照“理论分析—需求提取—系统设计—功能实现思路构建—效果评价”的技术路线展开，力求建立一套适用于水利水电施工建设场景的安全检查系统设计方案。首先，在理论分析阶段，围绕施工安全管理、安全检查机制和隐患排查治理理论展开研究。安全检查系统的核心目标并不只是实现信息录入，而是通过信息化手段优化安全管理流程，提高危险源识别能力和隐患整改效率。因此，在理论基础上，课题将结合事故致因理论、系统安全理论和闭环管理思想，对安全检查工作进行功能定位。事故的发生通常与人的不安全行为、物的不安全状态、环境因素以及管理缺陷有关，而安全检查工作正是识别这些问题并采取预防性措施的重要手段。基于这一理论，本课题将安全检查系统定义为集“检查、识别、记录、整改、反馈、统计”于一体的全过程管理工具。\\n其次，在需求提取阶段，对水利水电施工现场常见作业活动及其安全检查要点进行归纳。水利水电施工通常包括基坑开挖、边坡支护、混凝土浇筑、起重吊装、模板支架、高空作业、临时用电及施工机械运行等多个高风险环节。不同环节对应不同的检查标准和风险控制要求，因此系统设计必须体现检查对象的分类管理和检查项目的针对性设置。通过对现场管理流程的分析，可将系统使用者划分为管理员、安全员、检查人员、项目负责人等角色，并根据不同角色赋予差异化操作权限，以保证系统运行的规范性和可控性。\\n在系统设计阶段，课题将从系统架构设计、功能模块设计、数据库设计和业务流程设计四个方面展开。系统架构上，可采用前后端分离或传统分层架构思想，划分为表示层、业务逻辑层和数据层。表示层主要负责用户界面展示和操作交互；业务逻辑层负责安全检查业务规则处理，如隐患等级判定、整改期限控制、复查状态更新等；数据层负责各类信息的存储、查询与维护。功能模块方面，系统初步设计为五大模块：一是用户与权限管理模块，用于管理不同用户身份及操作范围；二是安全检查任务管理模块，用于制定检查计划、发布检查任务并记录检查结果；三是隐患整改管理模块，用于对发现的问题进行分类、定级、下发整改要求并追踪整改进度；四是统计分析模块，用于生成隐患数量、隐患类型分布、整改率、超期率等统计结果；五是系统维护模块，用于标准库维护、数据备份和参数设置等。数据库设计方面，围绕“用户表、检查表、隐患表、整改表、复查表、项目表”等核心数据实体建立逻辑关系，保证数据一致性与完整性。\\n在理论分析及计算部分，本课题虽然以系统设计为主，但仍可引入一定的数据统计与指标分析方法，用于支持系统功能合理性。例如，可设置隐患整改率=已完成整改隐患数/隐患总数×100%，隐患复查通过率=复查合格数/复查总数×100%，逾期整改率=逾期未整改隐患数/隐患总数×100%。通过这些指标计算，可以反映施工现场安全管理执行效果，也为后续统计分析模块的设计提供依据。若条件允许，还可对不同施工阶段的隐患分布进行对比分析，以判断系统在风险预警和管理改进方面的辅助价值。\\n在实验方法和步骤方面，本课题主要采用模拟业务场景验证的方式开展。第一步，整理水利水电施工安全检查业务流程和典型检查表内容，形成系统需求清单。第二步，根据需求清单绘制系统功能结构图、业务流程图和数据库E-R图，完成系统总体设计。第三步，选取典型场景，如日常巡检、专项检查、隐患整改通知、复查闭环等，对系统各模块进行流程模拟，验证功能衔接是否合理。第四步，通过构造测试数据，检验系统对检查记录录入、隐患信息分类、整改状态更新和统计报表生成等功能的支持情况。第五步，对系统设计中存在的问题进行修改优化，如功能重复、流程不清晰、字段设置不完整等。第六步，总结系统设计成果，并从实用性、规范性、可扩展性和安全管理效能提升等角度进行综合评价。\\n总体来看，本课题方案兼顾了安全工程专业背景下的风险管理需求和计算机软件类系统设计要求，能够较好体现理论联系实际的研究特点。通过该总体方案的实施，预期形成一套逻辑清晰、结构完整、适应水利水电施工现场安全管理需要的安全检查系统设计成果。\\n\\n三、项目进展、存在问题及解决办法。\\n1.毕设工作情况\\n本课题目前未更改题目，仍按照“水利水电施工建设中安全检查系统的设计”开展研究。开题以来，已按学校和学院要求推进毕业设计相关工作，围绕课题方向查阅了相关文献资料，并根据毕业论文写作规范逐步完善研究内容。周进展记录已按照要求进行整理，后续将继续及时补充并上传相关材料，确保过程管理资料完整规范。\\n2.课题研究进展\\n目前已基本完成课题前期调研和文献收集工作，对水利水电施工建设过程中的主要安全风险、安全检查流程及安全信息化管理需求有了较为系统的认识。通过查阅施工安全管理、安全检查系统设计、隐患排查治理机制等相关资料，已明确系统研究的基本思路和总体方向。在研究方法方面，已初步掌握文献分析、功能需求分析、系统模块划分及数据库逻辑设计等方法，并完成了系统功能框架的初步构思。现阶段已对系统涉及的主要业务流程进行了梳理，包括检查任务下达、现场检查记录、隐患上报、整改落实、复查销项和统计分析等内容。论文写作方面，已完成绪论部分初稿，并对第二章相关理论基础和研究方法部分进行了资料整理。\\n3. addressed to Peng Lin; celinpe@tsinghua.edu.cn\\nReceived 13 April 2014; Revised 18 June 2014; Accepted 24 June 2014; Published 9 July 2014\\nAcademic Editor: Xiao-Wei Ye\\n\\nAbstract\\nThe concern for workers’ safety in construction industry is reflected in many studies focusing on static safety risk identification and assessment. However, studies on real-time safety risk assessment aimed at reducing uncertainty and supporting quick response are rare. A method for real-time safety risk assessment (RTSRA) to implement a dynamic evaluation of worker safety states on construction site has been proposed in this paper. The method provides construction managers who are in charge of safety with more abundant information to reduce the uncertainty of the site. A quantitative calculation formula, integrating the influence of static and dynamic hazards and that of safety supervisors, is established to link the safety risk of workers with the locations of on-site assets. By employing the hidden Markov model (HMM), the RTSRA provides a mechanism for processing location data provided by the real-time location system (RTLS) and analyzing the probability distributions of different states in terms of false positives and negatives. Simulation analysis demonstrated the logic of the proposed method and how it works. Application case shows that the proposed RTSRA is both feasible and effective in managing construction project safety concerns.\\n1. Introduction\\nThe construction industry is one of the most dangerous industrial sectors worldwide [1, 2]. The working environment and the work tasks themselves are complex. The number of workers on site generally is large. Heavy machinery and a multitude of pipes, materials, and cables are always in evidence. In addition, sites are frequently not “tidy”; it is not surprising that both the fatality and badly injured accident rates on construction site are higher than those in other areas of work [3, 4]. According to the Bureau of Labor statistics in the United States [5], 4253 construction workers died on sites between 2008 and 2012. In China, the official statistics show that, in the one year, 2007, construction industrial accidents caused the deaths of 2722 workers [6]. Hydropower projects, in particular, have been found to have tougher working con- ditions than other construction sites and safety management of such construction projects is more difficult than that in any other segment of the industry [7]. Currently, China has been continuously developing large hydropower projects on main rivers, such as the Jinsha, Nujiang, and Yarlung\\nTsangpo River, investing large financial sums and manpower resources [8, 9]. Hence, without the improvement of the safety management, culture of the construction industry and the implementation of the national policy, which is “safety first and prevention oriented” [10] and which is also a vital management goal of hydropower enterprises in China, will not be achieved and it is unlikely that the accident rate on construction sites will decrease.\\nRecent studies related to construction safety manage- ment have asserted that most accidents on sites could have been reduced and some even eliminated, if there existed an effective and consistent safety management process of identification, planning, education/training, and inspection [11]. Safety risk assessment is very important for developing a safety management system [12–14]. Numerous risk assess- ment methods, such as safety checklists, fault tree analysis (FTA), and likelihood exposure consequences (LEC), have been developed to assist construction engineers and project managers in safety risk management [15]. The LEC method is widely used in casualty risk assessment in industrial construction [16]. Risk is considered to be the product ofthree factors: risk likelihood/probability (L), risk exposure frequency (E), and consequences (c). Casanovas et al. [17] presented a risk index which included 19 construction work activities and used the product of consequence and proba- bility to obtain the risk value of each activity. In addition, accident statistics, event tree analysis, and failure mode and effects analysis are also applied in safety management in the construction industry [18]. All the above methods, however, produce qualitative or semiqualitative analysis results. Such results cannot be considered reliable. For example, all of the three factors of LEC method are based on the subjec- tive evaluation of experts. Differences in the experience of different experts may lead to different evaluation results of the same risk [16]. In addition, most of these methods are passive approaches, heavily dependent on historical data and lacking timeliness [17, 19]. As a consequence, these methods, although producing results of interest and for guidance, cannot be used to quantitatively assess the current status of safety performance and the risk value.\\nIn order to operate a safety risk assessment procedure accurately, some quantitative methods based on probability theory and fuzzy sets theory are proposed. Using machine learning algorithms, Matı,as et al. [20] performed the identi- fication of hazardous sources on construction sites and the subsequent determination of the relationship between the accident and the cause. Matı,as et al. [21] designed a Bayesian network (BN) based method designed to analyze workplace accidents caused by falls from heights. Leu and Chang [22] successfully combined probabilistic methods with traditional methods (event tree analysis), enabling the setting up of a complete logic model to build a quantitative risk assessment method for construction projects. Regarding fuzzy set theory, Lee and Halpin [23] proposed a quantitative approach based on fuzzy logic for measuring the effects of accidents. Nunes [24] developed an expert system aimed at supporting risk analysis based on fuzzy sets. Azadeh et al. [25] developed a fuzzy expert system for performance assessment of health, safety, environment, and ergonomic system factors in a gas refinery with the objectives ofthe reduction of human error, creation of expert knowledge, and interpretation of large amounts of vague data. These quantitative methods based on probability and fuzzy sets theory offer relatively accu- rate evaluation results compared to those of the traditional methods and have been implemented in practical projects. Nevertheless, none of the above methods have been able to adapt to the dynamic context of construction projects, particularly those complex activities which contribute greatly to the safety risk of on-site workers. In addition, such methods cannot provide real-time information about safety states, which is a necessity to enable timely response and rescue.\\nRecently, extensive studies of real-time risk assessment have been carried out. An example is provided by the hidden Markov model (HMM) which is effective in many temporal and real-time pattern recognition applications, such as human action recognition [26, 27], speech recognition [28], and prediction of unknown events [29, 30]. Real-time risk assessment methods based on the HMM focus on the field of network security. These methods use the HMM toprocess the observation symbol sequences of invasions and attacks to determine the real-time safety risk of a network [31– 33]. In principle, the HMM based risk assessment method is accessible, but real-time safety risk assessment studies in construction management have not been found in the literature. The main reason for this could be the lack of availability of on-site asset real-time location information.As a consequence, some studies have focused on the development of intelligent control systems [34–37], adopting advanced communication technologies [38, 39], real-time location [40, 41], and intelligent algorithms as the means of developing new automated management and monitoring systems  [42–44]. However, these particular studies con- centrate more strongly on technical performances, such as accuracy of positioning and optimization of communi- cation frequency, while significant features such as safety management functions are not directly targeted. Geofencing which deals with the automatic detection of the entrance of individuals into dangerous preset forbidden zones is the basic safety management model revealed in the literature [39, 40]. But the simple trigger logic of the geofencing can only offer managers poor binary information concerning on- site workers’ behaviour (either within the forbidden zones or not in the forbidden zones) and may lead to frequent false positives. As a consequence, the development of an integrated method to perform real-time safety risk assessment of on- site workers based on probability concepts is necessary. The workers’ real-time location data should be provided by real-time location technologies. Overall, the development of automatic monitoring technology, especially the real-time location technology for individuals, provides great feasibility for implementation of real-time safety risk assessment. And the real-time location system for personnel and the moni- toring system for structure constitute a complete solution for health and safety management in industrial construction.\\nIn this study, GPS and Wi-Fi locating technology were chosen as the fundamental tracking tools. HMM was used as the risk assessment algorithm in constructing a RTSRA method for the safety management of large hydropower construction projects. The rest of this paper is organized as follows. Section 2 describes the construction of an assessment framework, including that of the real-time tracking system and the HMM modelling process. In Section 3, a simulation analysis is carried out on safety risk assessment process. Section 4 gives a preliminary application case of a dam construction site.\\n2. According to hazard kinetic characteristics, these risks can be divided into dynamic and static groups (Table 1).\\nMost dynamic hazards on the construction site are caused by vehicles, machinery, and risky tools. A static hazard andts affected area remain unchanged for a relatively long period. These latter hazards consist of elevation changes, hazardous installations, and other dangerous zones. For both dynamic and static hazards, the critical measurable factor is the distance between the workers and hazards [17]. Accidents are more likely to happen, if workers are close to hazards. A dissimilarity is that the position of a static hazard is fixed while a dynamic hazard is always moving. The distance between worker and safety supervisor can be obtained from the RTLS and hence is another measurable factor. 课题主要研究内容\n本课题围绕水利水电施工建设过程中的安全管理需求，设计一套适用于施工现场的安全检查系统。水利水电工程通常具有施工周期长、作业环境复杂、交叉作业频繁、高空作业与临时用电风险突出等特点，传统依赖人工巡查和纸质记录的安全检查方式存在信息滞后、隐患追踪困难、整改闭环不完善等问题。因此，本课题拟结合安全工程理论、施工现场风险管理需求以及信息化系统设计方法，对水利水电施工建设中的安全检查流程进行系统研究与优化。\n首先，将分析水利水电施工现场常见的安全风险类型，包括高处坠落、物体打击、机械伤害、触电、坍塌、有限空间作业风险等，并结合相关安全生产法律法规、行业标准和施工管理制度，明确安全检查系统应具备的基本功能和应用目标。其次，围绕施工单位、管理人员和检查人员的实际使用需求，对系统的功能模块进行划分，重点设计用户管理、隐患排查、检查任务发布、隐患整改跟踪、统计分析、信息查询和预警提示等功能。再次，在系统设计层面，课题将从总体架构、数据库结构、功能流程、界面逻辑等方面展开分析，形成较为完整的系统方案。\n最后，本课题还将通过业务流程梳理和模块设计，对系统运行过程中的数据流转关系进行分析，建立从“检查—记录—整改—复查—归档”的安全管理闭环机制。最后，结合水利水电施工场景，对所设计系统的适用性、可行性和预期应用效果进行评价，论证该系统在提升施工安全管理效率、规范隐患治理流程、降低事故风险方面的实际意义，为水利水电工程施工安全信息化建设提供一定参考。\n2、 课题与安全工程专业契合度描述\n本课题与安全工程专业具有较高的契合度，体现在研究对象、研究目的、研究方法以及成果应用等多个方面。安全工程专业的核心任务是识别危险源、分析事故致因、控制安全风险、预防生产事故，并通过工程技术和管理手段提升系统本质安全水平。本课题以水利水电施工建设中的安全检查系统设计为研究对象，本质上就是围绕施工安全风险防控和隐患治理机制优化展开研究，符合安全工程专业的培养方向和研究范畴。\n首先，从研究目标来看，本课题旨在解决施工现场安全检查信息化水平不足、隐患整改跟踪不及时、检查记录管理不规范等现实问题，这些问题直接关系到施工安全管理质量和事故预防效果。通过设计安全检查系统，可以强化隐患排查治理的闭环管理，提高风险识别与动态控制能力，这与安全工程专业强调的“预防为主、风险管控、持续改进”的理念高度一致。其次，从专业知识运用来看，课题需要综合运用安全系统工程、安全管理学、事故致因理论、危险源辨识、风险评价、职业健康安全管理等专业知识，对施工现场的风险特点和安全管理流程进行分析，并将这些理论要求转化为系统功能设计依据。\n再次，从工程应用角度看，水利水电工程属于高风险建设领域，安全管理要求高、涉及面广，是安全工程专业典型的应用场景之一。课题研究结果不仅能够服务于具体施工项目的安全检查管理，还能够为类似大型土木工程、基础设施建设项目的安全信息化管理提供借鉴。最后，从毕业论文培养目标来看，本课题兼具理论分析与工程实践特征，能够有效锻炼问题分析、方案设计、系统构建和专业表达等方面的综合能力，体现了安全工程专业“工程背景突出、管理技术结合、服务安全生产实际”的人才培养要求。因此，本课题在选题意义、研究内容、专业方法和应用价值上均与安全工程专业紧密相关。\n\n\t二、研究方案设计\n1、研究方法\n本课题采用文献研究法、系统分析法、需求分析法、功能模块设计法以及数据库设计方法相结合的研究思路，对水利水电施工建设中安全检查系统进行设计。首先，在课题研究初期，采用文献研究法查阅国内外关于施工安全管理、安全检查信息化系统、隐患排查治理机制以及水利水电工程施工安全控制等方面的研究成果，系统了解当前研究现状、常用技术路线及实际应用模式。通过对相关法律法规、国家标准、行业规范和已有研究文献的整理分析，明确系统设计必须满足的规范要求和业务逻辑基础。\n其次，采用系统分析法对水利水电施工现场安全检查业务进行分解，从管理对象、管理流程、风险要素、参与角色和信息传递路径等多个维度梳理系统运行机制。该方法有助于从整体角度把握安全检查工作的结构组成，避免功能设计碎片化，使系统能够覆盖检查任务制定、现场检查实施、隐患记录、整改通知、复查验收、统计汇总等完整流程。由于水利水电施工现场涉及建设单位、监理单位、施工单位及班组等多主体协同，系统分析法有利于明确各方权限边界与信息交互关系。\n再次，采用需求分析法确定系统功能。需求分析主要从实际应用出发，围绕安全管理人员、检查人员和项目负责人等不同用户群体的使用需求，分析他们在日常管理中面临的问题，如记录不统一、隐患难追踪、信息统计困难、整改时效性差等，并据此提炼出系统应具备的核心需求，包括账户权限管理、检查项目配置、隐患信息录入、整改通知下发、复查结果记录、报表统计分析和历史数据查询等功能。\n在系统实现思路方面，课题将采用功能模块设计法，将复杂系统划分为相对独立又相互关联的模块，包括基础信息管理模块、安全检查模块、隐患整改模块、统计分析模块和系统维护模块。通过模块化设计，不仅可以使系统结构更加清晰，便于后续功能扩展，也符合软件系统设计的基本原则。同时，在数据层面采用数据库设计方法，对用户信息、检查记录、隐患信息、整改记录、复查结果等关键数据进行逻辑建模，确保数据存储规范、查询高效、关联清晰。\n综合来看，这些方法的选择依据在于：一方面，本课题不仅需要符合安全工程专业对风险管理和事故预防的理论要求，也需要满足计算机软件类课题对系统逻辑性和可实现性的要求；另一方面，所选方法能够从理论支撑、业务分析、功能构建和数据组织四个层面共同服务于系统设计目标，保证研究结果既有专业性，也具有较强的工程应用价值。\n\n2、总体方案（包括理论分析及计算、实验方法和步骤）\n本课题总体方案按照“理论分析—需求提取—系统设计—功能实现思路构建—效果评价”的技术路线展开，力求建立一套适用于水利水电施工建设场景的安全检查系统设计方案。首先，在理论分析阶段，围绕施工安全管理、安全检查机制和隐患排查治理理论展开研究。安全检查系统的核心目标并不只是实现信息录入，而是通过信息化手段优化安全管理流程，提高危险源识别能力和隐患整改效率。因此，在理论基础上，课题将结合事故致因理论、系统安全理论和闭环管理思想，对安全检查工作进行功能定位。事故的发生通常与人的不安全行为、物的不安全状态、环境因素以及管理缺陷有关，而安全检查工作正是识别这些问题并采取预防性措施的重要手段。基于这一理论，本课题将安全检查系统定义为集“检查、识别、记录、整改、反馈、统计”于一体的全过程管理工具。\n其次，在需求提取阶段，对水利水电施工现场常见作业活动及其安全检查要点进行归纳。水利水电施工通常包括基坑开挖、边坡支护、混凝土浇筑、起重吊装、模板支架、高空作业、临时用电及施工机械运行等多个高风险环节。不同环节对应不同的检查标准和风险控制要求，因此系统设计必须体现检查对象的分类管理和检查项目的针对性设置。通过对现场管理流程的分析，可将系统使用者划分为管理员、安全员、检查人员、项目负责人等角色，并根据不同角色赋予差异化操作权限，以保证系统运行的规范性和可控性。\n在系统设计阶段，课题将从系统架构设计、功能模块设计、数据库设计和业务流程设计四个方面展开。系统架构上，可采用前后端分离或传统分层架构思想，划分为表示层、业务逻辑层和数据层。表示层主要负责用户界面展示和操作交互；业务逻辑层负责安全检查业务规则处理，如隐患等级判定、整改期限控制、复查状态更新等；数据层负责各类信息的存储、查询与维护。功能模块方面，系统初步设计为五大模块：一是用户与权限管理模块，用于管理不同用户身份及操作范围；二是安全检查任务管理模块，用于制定检查计划、发布检查任务并记录检查结果；三是隐患整改管理模块，用于对发现的问题进行分类、定级、下发整改要求并追踪整改进度；四是统计分析模块，用于生成隐患数量、隐患类型分布、整改率、超期率等统计结果；五是系统维护模块，用于标准库维护、数据备份和参数设置等。数据库设计方面，围绕“用户表、检查表、隐患表、整改表、复查表、项目表”等核心数据实体建立逻辑关系，保证数据一致性与完整性。\n在理论分析及计算部分，本课题虽然以系统设计为主，但仍可引入一定的数据统计与指标分析方法，用于支持系统功能合理性。例如，可设置隐患整改率=已完成整改隐患数/隐患总数×100%，隐患复查通过率=复查合格数/复查总数×100%，逾期整改率=逾期未整改隐患数/隐患总数×100%。通过这些指标计算，可以反映施工现场安全管理执行效果，也为后续统计分析模块的设计提供依据。若条件允许，还可对不同施工阶段的隐患分布进行对比分析，以判断系统在风险预警和管理改进方面的辅助价值。\n在实验方法和步骤方面，本课题主要采用模拟业务场景验证的方式开展。第一步，整理水利水电施工安全检查业务流程和典型检查表内容，形成系统需求清单。第二步，根据需求清单绘制系统功能结构图、业务流程图和数据库E-R图，完成系统总体设计。第三步，选取典型场景，如日常巡检、专项检查、隐患整改通知、复查闭环等，对系统各模块进行流程模拟，验证功能衔接是否合理。第四步，通过构造测试数据，检验系统对检查记录录入、隐患信息分类、整改状态更新和统计报表生成等功能的支持情况。第五步，对系统设计中存在的问题进行修改优化，如功能重复、流程不清晰、字段设置不完整等。第六步，总结系统设计成果，并从实用性、规范性、可扩展性和安全管理效能提升等角度进行综合评价。\n总体来看，本课题方案兼顾了安全工程专业背景下的风险管理需求和计算机软件类系统设计要求，能够较好体现理论联系实际的研究特点。通过该总体方案的实施，预期形成一套逻辑清晰、结构完整、适应水利水电施工现场安全管理需要的安全检查系统设计成果。\n\n三、项目进展、存在问题及解决办法。\n1.毕设工作情况\n本课题目前未更改题目，仍按照“水利水电施工建设中安全检查系统的设计”开展研究。开题以来，已按学校和学院要求推进毕业设计相关工作，围绕课题方向查阅了相关文献资料，并根据毕业论文写作规范逐步完善研究内容。周进展记录已按照要求进行整理，后续将继续及时补充并上传相关材料，确保过程管理资料完整规范。\n2.课题研究进展\n目前已基本完成课题前期调研和文献收集工作，对水利水电施工建设过程中的主要安全风险、安全检查流程及安全信息化管理需求有了较为系统的认识。通过查阅施工安全管理、安全检查系统设计、隐患排查治理机制等相关资料，已明确系统研究的基本思路和总体方向。在研究方法方面，已初步掌握文献分析、功能需求分析、系统模块划分及数据库逻辑设计等方法，并完成了系统功能框架的初步构思。现阶段已对系统涉及的主要业务流程进行了梳理，包括检查任务下达、现场检查记录、隐患上报、整改落实、复查销项和统计分析等内容。论文写作方面，已完成绪论部分初稿，并对第二章相关理论基础和研究方法部分进行了资料整理。\n3. Safety Risk Assessment Based on a Real-Time Location System for Hydropower Construction Sites\nAuthor information\nHanchen Jiang,1 Peng Lin,1 Qixiang Fan,2 and Maoshan Qiang1\n1         State Key Laboratory of Hydroscience and Engineering, Tsinghua University, Haidian, Beijing 100084, China\n2    China Yangtze Three Gorges Group Corporation, Beijing 100038, China\nCorrespondence should be addressed to Peng Lin; celinpe@tsinghua.edu.cn\nReceived 13 April 2014; Revised 18 June 2014; Accepted 24 June 2014; Published 9 July 2014\nAcademic Editor: Xiao-Wei Ye\n\nAbstract\nThe concern for workers’ safety in construction industry is reflected in many studies focusing on static safety risk identification and assessment. However, studies on real-time safety risk assessment aimed at reducing uncertainty and supporting quick response are rare. A method for real-time safety risk assessment (RTSRA) to implement a dynamic evaluation of worker safety states on construction site has been proposed in this paper. The method provides construction managers who are in charge of safety with more abundant information to reduce the uncertainty of the site. A quantitative calculation formula, integrating the influence of static and dynamic hazards and that of safety supervisors, is established to link the safety risk of workers with the locations of on-site assets. By employing the hidden Markov model (HMM), the RTSRA provides a mechanism for processing location data provided by the real-time location system (RTLS) and analyzing the probability distributions of different states in terms of false positives and negatives. Simulation analysis demonstrated the logic of the proposed method and how it works. Application case shows that the proposed RTSRA is both feasible and effective in managing construction project safety concerns.\n1. Introduction\nThe construction industry is one of the most dangerous industrial sectors worldwide [1, 2]. The working environment and the work tasks themselves are complex. The number of workers on site generally is large. Heavy machinery and a multitude of pipes, materials, and cables are always in evidence. In addition, sites are frequently not “tidy”; it is not surprising that both the fatality and badly injured accident rates on construction site are higher than those in other areas of work [3, 4]. According to the Bureau of Labor statistics in the United States [5], 4253 construction workers died on sites between 2008 and 2012. In China, the official statistics show that, in the one year, 2007, construction industrial accidents caused the deaths of 2722 workers [6]. Hydropower projects, in particular, have been found to have tougher working con- ditions than other construction sites and safety management of such construction projects is more difficult than that in any other segment of the industry [7]. Currently, China has been continuously developing large hydropower projects on main rivers, such as the Jinsha, Nujiang, and Yarlung\nTsangpo River, investing large financial sums and manpower resources [8, 9]. Hence, without the improvement of the safety management, culture of the construction industry and the implementation of the national policy, which is “safety first and prevention oriented” [10] and which is also a vital management goal of hydropower enterprises in China, will not be achieved and it is unlikely that the accident rate on construction sites will decrease.\nRecent studies related to construction safety manage- ment have asserted that most accidents on sites could have been reduced and some even eliminated, if there existed an effective and consistent safety management process of identification, planning, education/training, and inspection [11]. Safety risk assessment is very important for developing a safety management system [12–14]. Numerous risk assess- ment methods, such as safety checklists, fault tree analysis (FTA), and likelihood exposure consequences (LEC), have been developed to assist construction engineers and project managers in safety risk management [15]. The LEC method is widely used in casualty risk assessment in industrial construction [16]. Risk is considered to be the product ofthree factors: risk likelihood/probability (L), risk exposure frequency (E), and consequences (c). Casanovas et al. [17] presented a risk index which included 19 construction work activities and used the product of consequence and proba- bility to obtain the risk value of each activity. In addition, accident statistics, event tree analysis, and failure mode and effects analysis are also applied in safety management in the construction industry [18]. All the above methods, however, produce qualitative or semiqualitative analysis results. Such results cannot be considered reliable. For example, all of the three factors of LEC method are based on the subjec- tive evaluation of experts. Differences in the experience of different experts may lead to different evaluation results of the same risk [16]. In addition, most of these methods are passive approaches, heavily dependent on historical data and lacking timeliness [17, 19]. As a consequence, these methods, although producing results of interest and for guidance, cannot be used to quantitatively assess the current status of safety performance and the risk value.\nIn order to operate a safety risk assessment procedure accurately, some quantitative methods based on probability theory and fuzzy sets theory are proposed. Using machine learning algorithms, Matı,as et al. [20] performed the identi- fication of hazardous sources on construction sites and the subsequent determination of the relationship between the accident and the cause. Matı,as et al. [21] designed a Bayesian network (BN) based method designed to analyze workplace accidents caused by falls from heights. Leu and Chang [22] successfully combined probabilistic methods with traditional methods (event tree analysis), enabling the setting up of a complete logic model to build a quantitative risk assessment method for construction projects. Regarding fuzzy set theory, Lee and Halpin [23] proposed a quantitative approach based on fuzzy logic for measuring the effects of accidents. Nunes [24] developed an expert system aimed at supporting risk analysis based on fuzzy sets. Azadeh et al. [25] developed a fuzzy expert system for performance assessment of health, safety, environment, and ergonomic system factors in a gas refinery with the objectives ofthe reduction of human error, creation of expert knowledge, and interpretation of large amounts of vague data. These quantitative methods based on probability and fuzzy sets theory offer relatively accu- rate evaluation results compared to those of the traditional methods and have been implemented in practical projects. Nevertheless, none of the above methods have been able to adapt to the dynamic context of construction projects, particularly those complex activities which contribute greatly to the safety risk of on-site workers. In addition, such methods cannot provide real-time information about safety states, which is a necessity to enable timely response and rescue.\nRecently, extensive studies of real-time risk assessment have been carried out. An example is provided by the hidden Markov model (HMM) which is effective in many temporal and real-time pattern recognition applications, such as human action recognition [26, 27], speech recognition [28], and prediction of unknown events [29, 30]. Real-time risk assessment methods based on the HMM focus on the field of network security. These methods use the HMM toprocess the observation symbol sequences of invasions and attacks to determine the real-time safety risk of a network [31– 33]. In principle, the HMM based risk assessment method is accessible, but real-time safety risk assessment studies in construction management have not been found in the literature. The main reason for this could be the lack of availability of on-site asset real-time location information.As a consequence, some studies have focused on the development of intelligent control systems [34–37], adopting advanced communication technologies [38, 39], real-time location [40, 41], and intelligent algorithms as the means of developing new automated management and monitoring systems  [42–44]. However, these particular studies con- centrate more strongly on technical performances, such as accuracy of positioning and optimization of communi- cation frequency, while significant features such as safety management functions are not directly targeted. Geofencing which deals with the automatic detection of the entrance of individuals into dangerous preset forbidden zones is the basic safety management model revealed in the literature [39, 40]. But the simple trigger logic of the geofencing can only offer managers poor binary information concerning on- site workers’ behaviour (either within the forbidden zones or not in the forbidden zones) and may lead to frequent false positives. As a consequence, the development of an integrated method to perform real-time safety risk assessment of on- site workers based on probability concepts is necessary. The workers’ real-time location data should be provided by real-time location technologies. Overall, the development of automatic monitoring technology, especially the real-time location technology for individuals, provides great feasibility for implementation of real-time safety risk assessment. And the real-time location system for personnel and the moni- toring system for structure constitute a complete solution for health and safety management in industrial construction.\nIn this study, GPS and Wi-Fi locating technology were chosen as the fundamental tracking tools. HMM was used as the risk assessment algorithm in constructing a RTSRA method for the safety management of large hydropower construction projects. The rest of this paper is organized as follows. Section 2 describes the construction of an assessment framework, including that of the real-time tracking system and the HMM modelling process. In Section 3, a simulation analysis is carried out on safety risk assessment process. Section 4 gives a preliminary application case of a dam construction site.\n2. Unsafe worker activities are direct causes of accidents. Different unsafe activities are usually associated with specific hazards. It has been concluded by Casanovas et al.  [17] that there are 19 health and safety risks found in construction work associated with different activities. According to hazard kinetic characteristics, these risks can be divided into dynamic and static groups (Table 1).\nMost dynamic hazards on the construction site are caused by vehicles, machinery, and risky tools. A static hazard andts affected area remain unchanged for a relatively long period. These latter hazards consist of elevation changes, hazardous installations, and other dangerous zones. For both dynamic and static hazards, the critical measurable factor is the distance between the workers and hazards [17]. Accidents are more likely to happen, if workers are close to hazards. A dissimilarity is that the position of a static hazard is fixed while a dynamic hazard is always moving. In consequence, in addition to the tracking of workers, dynamic hazards must also be monitored by the RTLS in order to make valid safety risk assessments.\nAnother critical factor is the presence of safety supervi- sors. They are responsible for safety management and possess specialized safety knowledge [47]. Effective safety supervi- sion improves the safety performance of workers. The safety supervisor must be in such a position that he can have a posi- tive impact on a worker behaviour [48]. The distance between worker and safety supervisor can be obtained from the RTLS and hence is another measurable factor. 隐患整改率 = 已完成整改隐患数 / 隐患总数 × 100%\n- 复查通过率 = 复查合格数 / 复查总数 × 100%\n- 逾期整改率 = 逾期未整改隐患数 / 隐患总数 × 100%\n\n## 5. 异常与边界情况\n\n| 场景 | 处理方式 |\n|------|----------|\n| 检查人员未按时完成检查任务 | 系统自动标记任务逾期，并向安全员发送通知 |\n| 课题主要研究内容\n本课题围绕水利水电施工建设过程中的安全管理需求，设计一套适用于施工现场的安全检查系统。水利水电工程通常具有施工周期长、作业环境复杂、交叉作业频繁、高空作业与临时用电风险突出等特点，传统依赖人工巡查和纸质记录的安全检查方式存在信息滞后、隐患追踪困难、整改闭环不完善等问题。因此，本课题拟结合安全工程理论、施工现场风险管理需求以及信息化系统设计方法，对水利水电施工建设中的安全检查流程进行系统研究与优化。\n首先，将分析水利水电施工现场常见的安全风险类型，包括高处坠落、物体打击、机械伤害、触电、坍塌、有限空间作业风险等，并结合相关安全生产法律法规、行业标准和施工管理制度，明确安全检查系统应具备的基本功能和应用目标。其次，围绕施工单位、管理人员和检查人员的实际使用需求，对系统的功能模块进行划分，重点设计用户管理、隐患排查、检查任务发布、隐患整改跟踪、统计分析、信息查询和预警提示等功能。再次，在系统设计层面，课题将从总体架构、数据库结构、功能流程、界面逻辑等方面展开分析，形成较为完整的系统方案。\n最后，本课题还将通过业务流程梳理和模块设计，对系统运行过程中的数据流转关系进行分析，建立从“检查—记录—整改—复查—归档”的安全管理闭环机制。最后，结合水利水电施工场景，对所设计系统的适用性、可行性和预期应用效果进行评价，论证该系统在提升施工安全管理效率、规范隐患治理流程、降低事故风险方面的实际意义，为水利水电工程施工安全信息化建设提供一定参考。\n2、 课题与安全工程专业契合度描述\n本课题与安全工程专业具有较高的契合度，体现在研究对象、研究目的、研究方法以及成果应用等多个方面。安全工程专业的核心任务是识别危险源、分析事故致因、控制安全风险、预防生产事故，并通过工程技术和管理手段提升系统本质安全水平。本课题以水利水电施工建设中的安全检查系统设计为研究对象，本质上就是围绕施工安全风险防控和隐患治理机制优化展开研究，符合安全工程专业的培养方向和研究范畴。\n首先，从研究目标来看，本课题旨在解决施工现场安全检查信息化水平不足、隐患整改跟踪不及时、检查记录管理不规范等现实问题，这些问题直接关系到施工安全管理质量和事故预防效果。通过设计安全检查系统，可以强化隐患排查治理的闭环管理，提高风险识别与动态控制能力，这与安全工程专业强调的“预防为主、风险管控、持续改进”的理念高度一致。其次，从专业知识运用来看，课题需要综合运用安全系统工程、安全管理学、事故致因理论、危险源辨识、风险评价、职业健康安全管理等专业知识，对施工现场的风险特点和安全管理流程进行分析，并将这些理论要求转化为系统功能设计依据。\n再次，从工程应用角度看，水利水电工程属于高风险建设领域，安全管理要求高、涉及面广，是安全工程专业典型的应用场景之一。课题研究结果不仅能够服务于具体施工项目的安全检查管理，还能够为类似大型土木工程、基础设施建设项目的安全信息化管理提供借鉴。最后，从毕业论文培养目标来看，本课题兼具理论分析与工程实践特征，能够有效锻炼问题分析、方案设计、系统构建和专业表达等方面的综合能力，体现了安全工程专业“工程背景突出、管理技术结合、服务安全生产实际”的人才培养要求。因此，本课题在选题意义、研究内容、专业方法和应用价值上均与安全工程专业紧密相关。\n\n\t二、研究方案设计\n1、研究方法\n本课题采用文献研究法、系统分析法、需求分析法、功能模块设计法以及数据库设计方法相结合的研究思路，对水利水电施工建设中安全检查系统进行设计。首先，在课题研究初期，采用文献研究法查阅国内外关于施工安全管理、安全检查信息化系统、隐患排查治理机制以及水利水电工程施工安全控制等方面的研究成果，系统了解当前研究现状、常用技术路线及实际应用模式。通过对相关法律法规、国家标准、行业规范和已有研究文献的整理分析，明确系统设计必须满足的规范要求和业务逻辑基础。\n其次，采用系统分析法对水利水电施工现场安全检查业务进行分解，从管理对象、管理流程、风险要素、参与角色和信息传递路径等多个维度梳理系统运行机制。该方法有助于从整体角度把握安全检查工作的结构组成，避免功能设计碎片化，使系统能够覆盖检查任务制定、现场检查实施、隐患记录、整改通知、复查验收、统计汇总等完整流程。由于水利水电施工现场涉及建设单位、监理单位、施工单位及班组等多主体协同，系统分析法有利于明确各方权限边界与信息交互关系。\n再次，采用需求分析法确定系统功能。需求分析主要从实际应用出发，围绕安全管理人员、检查人员和项目负责人等不同用户群体的使用需求，分析他们在日常管理中面临的问题，如记录不统一、隐患难追踪、信息统计困难、整改时效性差等，并据此提炼出系统应具备的核心需求，包括账户权限管理、检查项目配置、隐患信息录入、整改通知下发、复查结果记录、报表统计分析和历史数据查询等功能。\n在系统实现思路方面，课题将采用功能模块设计法，将复杂系统划分为相对独立又相互关联的模块，包括基础信息管理模块、安全检查模块、隐患整改模块、统计分析模块和系统维护模块。通过模块化设计，不仅可以使系统结构更加清晰，便于后续功能扩展，也符合软件系统设计的基本原则。同时，在数据层面采用数据库设计方法，对用户信息、检查记录、隐患信息、整改记录、复查结果等关键数据进行逻辑建模，确保数据存储规范、查询高效、关联清晰。\n综合来看，这些方法的选择依据在于：一方面，本课题不仅需要符合安全工程专业对风险管理和事故预防的理论要求，也需要满足计算机软件类课题对系统逻辑性和可实现性的要求；另一方面，所选方法能够从理论支撑、业务分析、功能构建和数据组织四个层面共同服务于系统设计目标，保证研究结果既有专业性，也具有较强的工程应用价值。\n\n2、总体方案（包括理论分析及计算、实验方法和步骤）\n本课题总体方案按照“理论分析—需求提取—系统设计—功能实现思路构建—效果评价”的技术路线展开，力求建立一套适用于水利水电施工建设场景的安全检查系统设计方案。首先，在理论分析阶段，围绕施工安全管理、安全检查机制和隐患排查治理理论展开研究。安全检查系统的核心目标并不只是实现信息录入，而是通过信息化手段优化安全管理流程，提高危险源识别能力和隐患整改效率。因此，在理论基础上，课题将结合事故致因理论、系统安全理论和闭环管理思想，对安全检查工作进行功能定位。事故的发生通常与人的不安全行为、物的不安全状态、环境因素以及管理缺陷有关，而安全检查工作正是识别这些问题并采取预防性措施的重要手段。基于这一理论，本课题将安全检查系统定义为集“检查、识别、记录、整改、反馈、统计”于一体的全过程管理工具。\n其次，在需求提取阶段，对水利水电施工现场常见作业活动及其安全检查要点进行归纳。水利水电施工通常包括基坑开挖、边坡支护、混凝土浇筑、起重吊装、模板支架、高空作业、临时用电及施工机械运行等多个高风险环节。不同环节对应不同的检查标准和风险控制要求，因此系统设计必须体现检查对象的分类管理和检查项目的针对性设置。通过对现场管理流程的分析，可将系统使用者划分为管理员、安全员、检查人员、项目负责人等角色，并根据不同角色赋予差异化操作权限，以保证系统运行的规范性和可控性。\n在系统设计阶段，课题将从系统架构设计、功能模块设计、数据库设计和业务流程设计四个方面展开。系统架构上，可采用前后端分离或传统分层架构思想，划分为表示层、业务逻辑层和数据层。表示层主要负责用户界面展示和操作交互；业务逻辑层负责安全检查业务规则处理，如隐患等级判定、整改期限控制、复查状态更新等；数据层负责各类信息的存储、查询与维护。功能模块方面，系统初步设计为五大模块：一是用户与权限管理模块，用于管理不同用户身份及操作范围；二是安全检查任务管理模块，用于制定检查计划、发布检查任务并记录检查结果；三是隐患整改管理模块，用于对发现的问题进行分类、定级、下发整改要求并追踪整改进度；四是统计分析模块，用于生成隐患数量、隐患类型分布、整改率、超期率等统计结果；五是系统维护模块，用于标准库维护、数据备份和参数设置等。数据库设计方面，围绕“用户表、检查表、隐患表、整改表、复查表、项目表”等核心数据实体建立逻辑关系，保证数据一致性与完整性。\n在理论分析及计算部分，本课题虽然以系统设计为主，但仍可引入一定的数据统计与指标分析方法，用于支持系统功能合理性。例如，可设置隐患整改率=已完成整改隐患数/隐患总数×100%，隐患复查通过率=复查合格数/复查总数×100%，逾期整改率=逾期未整改隐患数/隐患总数×100%。通过这些指标计算，可以反映施工现场安全管理执行效果，也为后续统计分析模块的设计提供依据。若条件允许，还可对不同施工阶段的隐患分布进行对比分析，以判断系统在风险预警和管理改进方面的辅助价值。\n在实验方法和步骤方面，本课题主要采用模拟业务场景验证的方式开展。第一步，整理水利水电施工安全检查业务流程和典型检查表内容，形成系统需求清单。第二步，根据需求清单绘制系统功能结构图、业务流程图和数据库E-R图，完成系统总体设计。第三步，选取典型场景，如日常巡检、专项检查、隐患整改通知、复查闭环等，对系统各模块进行流程模拟，验证功能衔接是否合理。第四步，通过构造测试数据，检验系统对检查记录录入、隐患信息分类、整改状态更新和统计报表生成等功能的支持情况。第五步，对系统设计中存在的问题进行修改优化，如功能重复、流程不清晰、字段设置不完整等。第六步，总结系统设计成果，并从实用性、规范性、可扩展性和安全管理效能提升等角度进行综合评价。\n总体来看，本课题方案兼顾了安全工程专业背景下的风险管理需求和计算机软件类系统设计要求，能够较好体现理论联系实际的研究特点。通过该总体方案的实施，预期形成一套逻辑清晰、结构完整、适应水利水电施工现场安全管理需要的安全检查系统设计成果。\n\n三、项目进展、存在问题及解决办法。\n1.毕设工作情况\n本课题目前未更改题目，仍按照“水利水电施工建设中安全检查系统的设计”开展研究。开题以来，已按学校和学院要求推进毕业设计相关工作，围绕课题方向查阅了相关文献资料，并根据毕业论文写作规范逐步完善研究内容。周进展记录已按照要求进行整理，后续将继续及时补充并上传相关材料，确保过程管理资料完整规范。\n2.课题研究进展\n目前已基本完成课题前期调研和文献收集工作，对水利水电施工建设过程中的主要安全风险、安全检查流程及安全信息化管理需求有了较为系统的认识。通过查阅施工安全管理、安全检查系统设计、隐患排查治理机制等相关资料，已明确系统研究的基本思路和总体方向。在研究方法方面，已初步掌握文献分析、功能需求分析、系统模块划分及数据库逻辑设计等方法，并完成了系统功能框架的初步构思。现阶段已对系统涉及的主要业务流程进行了梳理，包括检查任务下达、现场检查记录、隐患上报、整改落实、复查销项和统计分析等内容。论文写作方面，已完成绪论部分初稿，并对第二章相关理论基础和研究方法部分进行了资料整理。\n3. workers’ safety in construction industry is reflected in many studies focusing on static safety risk identification and assessment. However, studies on real-time safety risk assessment aimed at reducing uncertainty and supporting quick response are rare. A method for real-time safety risk assessment (RTSRA) to implement a dynamic evaluation of worker safety states on construction site has been proposed in this paper. The method provides construction managers who are in charge of safety with more abundant information to reduce the uncertainty of the site. A quantitative calculation formula, integrating the influence of static and dynamic hazards and that of safety supervisors, is established to link the safety risk of workers with the locations of on-site assets. By employing the hidden Markov model (HMM), the RTSRA provides a mechanism for processing location data provided by the real-time location system (RTLS) and analyzing the probability distributions of different states in terms of false positives and negatives. Simulation analysis demonstrated the logic of the proposed method and how it works. Application case shows that the proposed RTSRA is both feasible and effective in managing construction project safety concerns.\n1. Introduction\nThe construction industry is one of the most dangerous industrial sectors worldwide [1, 2]. The working environment and the work tasks themselves are complex. The number of workers on site generally is large. Heavy machinery and a multitude of pipes, materials, and cables are always in evidence. In addition, sites are frequently not “tidy”; it is not surprising that both the fatality and badly injured accident rates on construction site are higher than those in other areas of work [3, 4]. According to the Bureau of Labor statistics in the United States [5], 4253 construction workers died on sites between 2008 and 2012. In China, the official statistics show that, in the one year, 2007, construction industrial accidents caused the deaths of 2722 workers [6]. Hydropower projects, in particular, have been found to have tougher working con- ditions than other construction sites and safety management of such construction projects is more difficult than that in any other segment of the industry [7]. Currently, China has been continuously developing large hydropower projects on main rivers, such as the Jinsha, Nujiang, and Yarlung\nTsangpo River, investing large financial sums and manpower resources [8, 9]. Hence, without the improvement of the safety management, culture of the construction industry and the implementation of the national policy, which is “safety first and prevention oriented” [10] and which is also a vital management goal of hydropower enterprises in China, will not be achieved and it is unlikely that the accident rate on construction sites will decrease.\nRecent studies related to construction safety manage- ment have asserted that most accidents on sites could have been reduced and some even eliminated, if there existed an effective and consistent safety management process of identification, planning, education/training, and inspection [11]. Safety risk assessment is very important for developing a safety management system [12–14]. Numerous risk assess- ment methods, such as safety checklists, fault tree analysis (FTA), and likelihood exposure consequences (LEC), have been developed to assist construction engineers and project managers in safety risk management [15]. The LEC method is widely used in casualty risk assessment in industrial construction [16]. Risk is considered to be the product ofthree factors: risk likelihood/probability (L), risk exposure frequency (E), and consequences (c). Casanovas et al. [17] presented a risk index which included 19 construction work activities and used the product of consequence and proba- bility to obtain the risk value of each activity. In addition, accident statistics, event tree analysis, and failure mode and effects analysis are also applied in safety management in the construction industry [18]. All the above methods, however, produce qualitative or semiqualitative analysis results. Such results cannot be considered reliable. For example, all of the three factors of LEC method are based on the subjec- tive evaluation of experts. Differences in the experience of different experts may lead to different evaluation results of the same risk [16]. In addition, most of these methods are passive approaches, heavily dependent on historical data and lacking timeliness [17, 19]. As a consequence, these methods, although producing results of interest and for guidance, cannot be used to quantitatively assess the current status of safety performance and the risk value.\nIn order to operate a safety risk assessment procedure accurately, some quantitative methods based on probability theory and fuzzy sets theory are proposed. Using machine learning algorithms, Matı,as et al. [20] performed the identi- fication of hazardous sources on construction sites and the subsequent determination of the relationship between the accident and the cause. Matı,as et al. [21] designed a Bayesian network (BN) based method designed to analyze workplace accidents caused by falls from heights. Leu and Chang [22] successfully combined probabilistic methods with traditional methods (event tree analysis), enabling the setting up of a complete logic model to build a quantitative risk assessment method for construction projects. Regarding fuzzy set theory, Lee and Halpin [23] proposed a quantitative approach based on fuzzy logic for measuring the effects of accidents. Nunes [24] developed an expert system aimed at supporting risk analysis based on fuzzy sets. Azadeh et al. [25] developed a fuzzy expert system for performance assessment of health, safety, environment, and ergonomic system factors in a gas refinery with the objectives ofthe reduction of human error, creation of expert knowledge, and interpretation of large amounts of vague data. These quantitative methods based on probability and fuzzy sets theory offer relatively accu- rate evaluation results compared to those of the traditional methods and have been implemented in practical projects. Nevertheless, none of the above methods have been able to adapt to the dynamic context of construction projects, particularly those complex activities which contribute greatly to the safety risk of on-site workers. In addition, such methods cannot provide real-time information about safety states, which is a necessity to enable timely response and rescue.\nRecently, extensive studies of real-time risk assessment have been carried out. An example is provided by the hidden Markov model (HMM) which is effective in many temporal and real-time pattern recognition applications, such as human action recognition [26, 27], speech recognition [28], and prediction of unknown events [29, 30]. Real-time risk assessment methods based on the HMM focus on the field of network security. These methods use the HMM toprocess the observation symbol sequences of invasions and attacks to determine the real-time safety risk of a network [31– 33]. In principle, the HMM based risk assessment method is accessible, but real-time safety risk assessment studies in construction management have not been found in the literature. The main reason for this could be the lack of availability of on-site asset real-time location information.As a consequence, some studies have focused on the development of intelligent control systems [34–37], adopting advanced communication technologies [38, 39], real-time location [40, 41], and intelligent algorithms as the means of developing new automated management and monitoring systems  [42–44]. However, these particular studies con- centrate more strongly on technical performances, such as accuracy of positioning and optimization of communi- cation frequency, while significant features such as safety management functions are not directly targeted. Geofencing which deals with the automatic detection of the entrance of individuals into dangerous preset forbidden zones is the basic safety management model revealed in the literature [39, 40]. But the simple trigger logic of the geofencing can only offer managers poor binary information concerning on- site workers’ behaviour (either within the forbidden zones or not in the forbidden zones) and may lead to frequent false positives. As a consequence, the development of an integrated method to perform real-time safety risk assessment of on- site workers based on probability concepts is necessary. The workers’ real-time location data should be provided by real-time location technologies. Overall, the development of automatic monitoring technology, especially the real-time location technology for individuals, provides great feasibility for implementation of real-time safety risk assessment. And the real-time location system for personnel and the moni- toring system for structure constitute a complete solution for health and safety management in industrial construction.\nIn this study, GPS and Wi-Fi locating technology were chosen as the fundamental tracking tools. HMM was used as the risk assessment algorithm in constructing a RTSRA method for the safety management of large hydropower construction projects. The rest of this paper is organized as follows. Section 2 describes the construction of an assessment framework, including that of the real-time tracking system and the HMM modelling process. In Section 3, a simulation analysis is carried out on safety risk assessment process. Section 4 gives a preliminary application case of a dam construction site.\n2. Unsafe worker activities are direct causes of accidents. Different unsafe activities are usually associated with specific hazards. It has been concluded by Casanovas et al.  [17] that there are 19 health and safety risks found in construction work associated with different activities. According to hazard kinetic characteristics, these risks can be divided into dynamic and static groups (Table 1).\nMost dynamic hazards on the construction site are caused by vehicles, machinery, and risky tools. A static hazard andts affected area remain unchanged for a relatively long period. These latter hazards consist of elevation changes, hazardous installations, and other dangerous zones. For both dynamic and static hazards, the critical measurable factor is the distance between the workers and hazards [17]. Accidents are more likely to happen, if workers are close to hazards. A dissimilarity is that the position of a static hazard is fixed while a dynamic hazard is always moving. In consequence, in addition to the tracking of workers, dynamic hazards must also be monitored by the RTLS in order to make valid safety risk assessments.\nAnother critical factor is the presence of safety supervi- sors. They are responsible for safety management and possess specialized safety knowledge [47]. Effective safety supervi- sion improves the safety performance of workers. The safety supervisor must be in such a position that he can have a posi- tive impact on a worker behaviour [48]. The distance between worker and safety supervisor can be obtained from the RTLS and hence is another measurable factor. export interface Profile {
   [key: string]: unknown;
}

export async function getProfile(userId: string): Promise<Profile | null> {
  const {data, error} = await supabase.from('profiles').select('*').eq('id', userId).maybeSingle()

  if (error) {
    console.error('Failed to fetch user profile:', error)
    return null
  }
  return data
}

interface AuthContextType {
  user: User | null
  profile: Profile | null
  loading: boolean
  signInWithUsername: (username: string, password: string) => Promise<{error: Error | null}>
  signUpWithUsername: (username: string, password: string) => Promise<{error: Error | null}>
  signUpWithPhone: (phone: string, password: string) => Promise<{error: Error | null}>
  signInWithPhone: (phone: string) => Promise<{error: Error | null}>
  verifyPhoneOtp: (phone: string, code: string) => Promise<{error: Error | null}>
  signInWithWechat: () => Promise<{error: Error | null}>
  signOut: () => Promise<void>
  refreshProfile: () => Promise<void>
}

const AuthContext = createContext<AuthContextType | undefined>(undefined)

export function AuthProvider({children}: {children: ReactNode}) {
  const [user, setUser] = useState<User | null>(null)
  const [profile, setProfile] = useState<Profile | null>(null)
  const [loading, setLoading] = useState(true)

  const refreshProfile = async () => {
    if (!user) {
      setProfile(null)
      return
    }

    const profileData = await getProfile(user.id)
    setProfile(profileData)
  }

  useEffect(() => {
    supabase.auth
      .getSession()
      .then(({data: {session}}) => {
        setUser(session?.user ?? null)
        if (session?.user) {
          getProfile(session.user.id).then(setProfile)
        }
        setLoading(false)
      })
      .catch((error) => {
        console.warn('Failed to get session:', error)
        setUser(null)
        setProfile(null)
        setLoading(false)
      })

    // In this function, do NOT use any await calls. Use `.then()` instead to avoid deadlocks.
    const {
      data: {subscription}
    } = supabase.auth.onAuthStateChange((_event, session) => {
      setUser(session?.user ?? null)
      if (session?.user) {
        getProfile(session.user.id).then(setProfile)
      } else {
        setProfile(null)
      }
    })

    return () => subscription.unsubscribe()
  }, [])

  const signInWithUsername = async (username: string, password: string) => {
    try {
      const email = `${username}@miaoda.com`
      const {error} = await supabase.auth.signInWithPassword({
        email,
        password
      })

      if (error) throw error
      return {error: null}
    } catch (error) {
      return {error: error as Error}
    }
  }

  const tabBarPages = getTabBarPages()

// Throttled navigation to prevent duplicate redirects
let isNavigating = false
function navigateToLogin(currentPath: string): void {
  if (isNavigating) {
    return
  }

  isNavigating = true

  // Save current path for redirect after login
  Taro.setStorageSync(STORAGE_KEY_REDIRECT_PATH, currentPath)
  const navigateMethod = isTabBarPage(currentPath) ? Taro.navigateTo : Taro.redirectTo
  navigateMethod({url: LOGIN_PAGE_PATH})

  // Reset flag after 100ms
  setTimeout(() => {

  }, 100)

/**
 * Route guard component for authentication protection
 * @warning DO NOT use this component in app.tsx! Wrap pages with withRouteGuard HOC
 */
function RouteGuard({children}: {children: React.ReactNode}) {
  const {user, loading} = useAuth()
  const [shouldRender, setShouldRender] = useState(false)

  const checkAuth = useCallback(() => {
    if (loading) {
      setShouldRender(false)
      return
    }

    const currentPath: string = Taro.getCurrentInstance()?.router?.path || ''

    // Allow access if user is authenticated or page is public
    const isPublic = PUBLIC_PAGE_PATHS.some((publicPath) => currentPath?.includes(publicPath))
    if (user || isPublic) {
      setShouldRender(true)
      return
    }
    if (currentPath && !currentPath?.includes(LOGIN_PAGE_PATH)) {
      navigateToLogin(currentPath)
      setShouldRender(false)
      return
    }
    setShouldRender(false)
  }, [user, loading])

  // Check auth when page is shown (handles tab switching)
  useDidShow(() => {
    checkAuth()
  })

  // Check auth when component mounts or auth state changes
  useEffect(() => {
    checkAuth()
  }, [checkAuth])

  if (!shouldRender) {
    return null
  }

  return <>{children}</>
}
